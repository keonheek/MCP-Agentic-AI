# SDC 리소스 3: 컨설팅 슬라이드 구조

---

## 피라미드 원칙 (Pyramid Principle)

맥킨지 Barbara Minto가 만든 컨설팅의 핵심 커뮤니케이션 원칙:

**결론을 먼저, 근거를 나중에.**

일반적인 발표:
```
배경 설명 → 분석 과정 → 데이터 → 결론
```

컨설팅 발표:
```
결론 (So What?) → 근거 1 → 근거 2 → 근거 3
```

**이유:** 임원은 바쁩니다. 30초 안에 핵심을 파악하고 싶어합니다. 학교 리포트처럼 "서론-본론-결론" 순서로 쓰면 핵심이 마지막에 나오기 때문에 비효율적입니다.

---

## 슬라이드 제목 = 인사이트

컨설팅 슬라이드에서 제목은 단순한 라벨이 아니라 **그 슬라이드의 결론**입니다.

| 나쁜 제목 (라벨형) | 좋은 제목 (인사이트형) |
|--------------------|-----------------------|
| 시장 분석 | IT 컨설팅 시장은 AI 수요로 연 12% 성장 |
| 경쟁사 현황 | 국내 3대 컨설팅사 모두 AI 팀 신설 (2025) |
| 리서치 결과 | 대학생 컨설팅 학회의 차별화 포인트는 실행력 |
| 결론 | SDC는 IT/AI 컨설팅 역량을 1순위로 키워야 함 |

**테스트:** 제목만 읽고 슬라이드 내용을 파악할 수 있으면 좋은 제목.

---

## MECE 프레임워크

**M**utually **E**xclusive, **C**ollectively **E**xhaustive
(상호 배타적이고 전체를 포괄하는)

정보를 분류할 때 겹치지 않으면서 빠짐없이 나누는 것.

예시 — "컨설팅의 가치"를 분류:

나쁜 분류 (겹침):
- 전략 수립
- 의사결정 지원 ← 전략 수립과 겹침
- 실행 지원

좋은 분류 (MECE):
- 전략 (What to do)
- 운영 (How to do it)
- 기술 (What tools to use)

## 5-7장 슬라이드 흐름 (이번 과제)

```
슬라이드 1: 표지
  → 팀명, 주제, 날짜

슬라이드 2: 컨설팅의 정의와 역할
  → 제목 예시: "컨설팅은 기업의 문제를 구조화하고 해결하는 전문 서비스"
  → 본문: 핵심 정의 + 주요 유형 (전략/운영/IT)
  → 시각화: 유형별 간단한 비교표

슬라이드 3: IT 컨설팅 — 일반 컨설팅과의 차이
  → 제목 예시: "IT 컨설팅은 기술을 비즈니스 전략과 연결하는 것"
  → 본문: IT 컨설팅만의 특징 2-3가지
  → 시각화: 비교 다이어그램

슬라이드 4: 리서치 인사이트 #1
  → 제목 = 가장 중요한 발견 (인사이트형 문장)
  → 근거: 데이터 + 출처
  → "So What?": SDC에 주는 시사점

슬라이드 5: 리서치 인사이트 #2
  → 두 번째 핵심 발견
  → 같은 구조

슬라이드 6: SDC에 적용할 점
  → 제목 예시: "SDC는 [X]를 통해 실무 역량을 높일 수 있다"
  → 구체적 행동 방안 3가지 (번호 매기기)
  → 각 행동에 기대 효과 한 줄

슬라이드 7: 요약 + 다음 단계
  → 핵심 메시지 1-2문장
  → 다음에 할 일 (구체적)
```

## 슬라이드 리뷰 체크리스트

발표 전에 모든 슬라이드를 아래 기준으로 점검하세요:

- [ ] 제목이 인사이트형 문장인가? (라벨이 아닌지)
- [ ] 한 슬라이드에 메시지가 1개뿐인가?
- [ ] 데이터에 출처가 있는가?
- [ ] 텍스트가 3줄 이내인가?
- [ ] "So What?"에 답할 수 있는가?
- [ ] 폰트 크기가 18pt 이상인가?
- [ ] 오타/맞춤법 확인했는가?
