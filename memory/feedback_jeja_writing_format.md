## Notion Page Structure

Toggle headings with color-coded backgrounds:
- 기도/성경읽기: blue_bg
- 성경대학: purple_bg
- 설교: green_bg
- 교재 과제: yellow_bg
- Assignment: orange_bg

---

## 관찰 (Observation)

Bullet list. Each item:
- States the observation plainly
- Second sentence adds the interpretive note or Hebrew/structural insight — NO em-dashes anywhere

Example:
`- 룻이 룻기에서 처음으로 입을 열어 말한다. 룻의 첫 발언은 가장 강력한 언약적 헌신의 선언이다. 가는 곳, 머무는 곳, 백성, 하나님, 죽음, 무덤까지 여섯 겹을 동일시한다.`

---

## 해석 (Interpretation)

Each section:
1. `<span color="brown_bg">**\[절범위 — 제목\]**</span>` header
2. Verse callout block (`<callout icon="📖" color="gray_bg">`)
3. 육하원칙 table (`<table header-row="true">`) — 언제인가? / 누가? / 어디에서? / 어떤 행동을 했는가? / 무슨 일이 있었는가?
4. Analysis paragraph — Hebrew word studies, intertextual links, theological significance

---

## 느낀점 (Personal Reflection)

Structure: 2-3 paragraphs
- Para 1: What hit hardest — leads with the most striking insight from the passage
- Para 2: Secondary observation — a tension, paradox, or character noticed
- Para 3 (optional): Compassion/community angle — how others in the text respond

Ends with:
- **개인 이야기 연결을 위한 질문 (건희님이 채워주세요)** — 3 questions, each ending in `인가?`
- **적용**: one concrete, present-tense action for the week

---

## 교재 과제 — Full section order

1. Full verse callout (`<callout icon="📑" color="gray_bg">`) — ALL verses in passage
2. `## 관찰` — bullet list
3. `## 해석` — per-절범위 sections with callout + table + analysis
4. `## 느낀점` — paragraphs + questions
5. `## 적용` — one action

---

## Assignment Toggle (jwj.kr plain text)

Rules:
- NO tables, callouts, bold, span tags, or toggles
- Section labels: `[섹션 이름]` in square brackets on its own line
- 절 labels: `[1:15-16절 — 제목]` on its own line
- 육하원칙: labeled lines — `언제:` / `누가:` / `어디에서:` / `어떤 행동:` / `무슨 일:`
- 느낀점: paragraphs as-is (plain text)
- 결단: leave as blank `결단:` — Keonhee fills in manually before submitting

---

## Writing Style Notes

- **NO em-dashes anywhere** — not in 관찰, 해석, 느낀점, or Assignment. Use a new sentence instead.
- **No AI-fishy phrases:** avoid "핵심은 이것이다:", "동시에", "사실 그 언어 자체가", "즉,", "결국", or any phrasing that reads like a summarizing AI
- 느낀점 opens with what hit hardest, NOT a summary
- Personal and honest — admits prior wrong assumptions, then corrects
- No duplicate sentences — each point said once, clearly
- Question endings: `인가?` not `나요?`
- 결단: one concrete, time-bound action. Never pre-fill in Assignment toggle — always leave blank for Keonhee

**Why:** 4주차 edits — Keonhee flagged em-dashes and AI-fishy writing as things to remove. The goal is text that reads like a real person wrote it.

**How to apply:** Every time /jeja generates any content — 관찰, 해석, 느낀점, Assignment — zero em-dashes, zero AI summarizing phrases. Write plainly.
