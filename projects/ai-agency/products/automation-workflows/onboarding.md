# 온보딩 가이드

_추가 자동화 플로우 도입 시 클라이언트 전달 문서_

---

## 필요 정보 (계약 후 요청)

| 항목 | 예시 | 용도 |
|---|---|---|
| 브랜드명 | 스킨로직 | 메시지 내 브랜드 표기 |
| 카카오톡 채널 | @skinlogic | 발송 채널 |
| 대표 연락처 | 010-XXXX-XXXX | 부정 후기 에스컬레이션 |
| 리뷰 페이지 URL | brand.com/review | 리뷰 요청 메시지 링크 |
| 인스타그램 계정 | @skinlogic_kr | IG 리포스트 요청 |
| 플랫폼 | 카페24 / 아임웹 / 스마트스토어 | 연동 방식 결정 |
| 할인 코드 생성 권한 | (관리자 계정 공유 또는 CSV) | 자동 코드 발급용 |

---

## 플랫폼별 연동

### 카페24
1. [쇼핑몰 설정] -> [Webhook 설정]
2. 이벤트: 장바구니 이탈, 주문 완료, 배송 완료, 재입고
3. URL: 1stmover 제공 엔드포인트 입력

### 아임웹
1. [설정] -> [API/Webhook]
2. 이벤트 선택 후 URL 입력, 인증 토큰 공유

### 스마트스토어
- 공식 Webhook 미지원 (API 폴링 방식)
- 스마트스토어 센터 -> API 키 발급 후 공유

---

## 발송 전 테스트

1. 테스트 고객 데이터 1건 입력
2. 각 플로우 트리거 수동 실행
3. 대표님 연락처로 테스트 발송 확인
4. 승인 후 정식 운영 전환

---

## 운영 시작 후

- 주 1회: 발송 현황 리포트
- 월 1회: 플로우별 전환율 리포트
- 이슈 발생: 4시간 이내 연락 (평일)
