# Demo Script: 학원 수강료 결제 자동화

대상: 학원 원장 (수강생 20명 이상, 수동 결제 확인 중)
소요 시간: 10분
목표: STANDARD 계약 (₩400K 설치 + ₩100K/월)

---

## 사전 준비

- [ ] demo_billing.py 실행 확인
- [ ] Supabase 대시보드 링크 준비
- [ ] 사이다페이 vs 1stmover 비용 비교표 준비

---

## 오프닝 (2분)

"원장님, 매달 수강료 확인하시는 데 얼마나 시간이 드세요?" (답변 듣기)

"저희 데이터로는 수강생 30명 기준 월 초 확인에 2-3시간, 미납 독촉에 추가 1시간이 소요됩니다. 오늘 10분 안에 이걸 어떻게 없애는지 보여드리겠습니다."

---

## 데모 단계 1: 자동 결제 시뮬레이션 (4분)

```bash
python demo/demo_billing.py --students 5 --date 2026-06-05 --simulate
```

"5명의 학생 수강료가 자동으로 처리됩니다. 성공 4명, 실패 1명으로 설정해뒀습니다."

(출력 확인)

"성공한 4명 학부모에게는 자동으로 알림톡이 갔고, 실패한 1명은 원장님 카카오로 즉시 알림이 갔습니다. 원장님이 아무것도 안 하셔도 됩니다."

---

## 데모 단계 2: 미납 독촉 자동화 (2분)

```bash
python demo/demo_billing.py --mode dunning --student "테스트 학생" --days_overdue 3
```

"3일 후에 자동으로 이런 독촉 알림톡이 갑니다. 직접 연락 안 하셔도 됩니다. 말투도 원장님 스타일로 맞출 수 있습니다."

---

## 데모 단계 3: Supabase 대시보드 (2분)

Supabase 링크 열기

"전체 결제 이력이 여기 실시간으로 기록됩니다. 누가 냈는지, 누가 안 냈는지, 총 수령액이 얼마인지 한눈에 보입니다."

---

## 비용 비교 (2분)

"사이다페이 쓰고 계시면: 수강생 [N]명 x ₩[N]K 수강료 x 1.5% = 월 ₩[N]K 수수료"

"1stmover: 정액 ₩100K/월. 수강생이 늘어도 비용 고정입니다."

"설치비 ₩400K는 보통 3개월이면 회수됩니다."
