## 1단계 - 훅 (5분): "이미 위반 중입니다"

**스크립트:**

"[브랜드명]에서 Claude API 또는 ChatGPT를 쓰고 계신가요?
그럼 고객 상담 내용이 지금 미국 서버로 전송되고 있는 겁니다.

9월 11일부터 이걸 개인정보처리방침에 명시하고 동의를 받지 않으면
항목당 최대 3천만 원 과태료 대상입니다.

피부 상태나 알레르기 정보를 수집하고 계신 경우에는
고위험 침해 시 매출의 10%까지 과징금이 부과됩니다."

**행동:** 브라우저에서 브랜드 개인정보처리방침 열기 → 국외 이전 조항 없는 것 직접 확인

## 2단계 - 리스크 어세스먼트 결과 공유 (5분)

미리 작성한 1페이지 PIPA 리스크 어세스먼트 출력본 제시.

**스크립트:**

"저번에 보내드렸던 체크리스트 결과입니다.
High 리스크가 [N]개 나왔어요.
특히 이 두 가지가 즉시 대응이 필요합니다:
1. AI API 국외 이전 고지 없음
2. 침해 72시간 신고 절차 없음

9월 11일까지 [X]일 남았습니다."

D-day 카운트다운, 조문별 현황, 열람/삭제 요청 추적이 한눈에 보입니다.

빨간 불이 켜진 항목이 지금 당장 과태료 대상 항목입니다.
저희가 납품하면 이 화면을 실시간으로 업데이트해 드립니다."

**포인트:** 제28조의2 빨간 불과 제34조 빨간 불 클로즈업

## 4단계 - 침해 대응 드릴 시연 (8분)

터미널에서 실행: `python demo/demo_incident_drill.py`

**스크립트:**

"이게 실제 침해가 발생했을 때 작동하는 자동화 대응 시스템입니다.
[터미널 실행]

지금 보시는 게 화장품 D2C 브랜드 CRM에 비인가 접근이 발생한 시나리오입니다.
- 심각도 자동 판정: HIGH
- 72시간 PIPC 신고 기한 자동 계산
- PIPC 신고서 자동 생성
- 2,450명 SMS 통지문 자동 생성

이 72시간을 혼자 대응하시려면... 솔직히 쉽지 않습니다.
저희가 즉시 대응 착수합니다."

## 자주 나오는 질문 (Q&A)

**Q. 변호사가 아닌데 법적 효력이 있나요?**
"저희는 법률 자문이 아닌 운영 실무를 지원합니다.
DPA 초안과 처리방침은 법무 법인 검토를 받으시도록 연계해 드릴 수 있습니다.
다만 99%의 소규모 브랜드는 법무 법인이 요구하는 수준의 준비도 안 되어 있습니다.
저희가 그 기초를 잡아드리는 겁니다."

**Q. 저희 규모가 작아서 단속 대상이 될까요?**
"PIPC는 2026년부터 AI API 사용 사업자를 집중 점검한다고 발표했습니다.
규모보다 피부/건강 데이터 수집 여부가 우선 기준입니다.
한 건의 침해 신고로 전수 조사가 시작될 수 있습니다."

**Q. Make.com, Mailchimp는 꼭 고지해야 하나요?**
"네. 국외 이전 고지 의무는 서비스 규모와 무관합니다.
Make.com은 EU, Mailchimp는 미국 서버입니다.
둘 다 명시적 동의와 처리방침 공개가 필요합니다."
