# SLA (서비스 수준 협약)

---

## 가용성

| 항목 | 기준 |
|---|---|
| 월간 서비스 가용성 | 99.0% 이상 |
| 계획된 점검 시간 | 월 최대 2시간 (새벽 2시-4시, 사전 3일 공지) |
| 비계획 장애 허용 시간 | 월 7.3시간 이내 |

당사는 자체 인프라 가동률 99.9% SLA를 보장합니다. 클라이언트의 비즈니스가 우리 서버에 의존하기에.

---

## 응답 시간

| 장애 등급 | 정의 | 초기 응답 | 해결 목표 |
|---|---|---|---|
| P1 (긴급) | 전체 자동화 흐름 중단 | 1시간 이내 | 4시간 이내 |
| P2 (높음) | 특정 플랫폼 연동 오류 | 4시간 이내 | 24시간 이내 |
| P3 (일반) | 부분 기능 오류, 성능 저하 | 1 영업일 이내 | 3 영업일 이내 |
| P4 (낮음) | 개선 요청, 비긴급 문의 | 2 영업일 이내 | 협의 |

영업시간: 평일 09:00-18:00 (공휴일 제외).

---

## 플랫폼별 주요 장애 대응

| 플랫폼 | 주요 장애 유형 | 대응 방식 |
|---|---|---|
| Cafe24 | API 버전 변경, 토큰 만료 | 자동 토큰 갱신, 버전 핀 고정 |
| Imweb | 24시간 토큰 만료 | 자동 재인증 |
| 스마트스토어 | 30분 토큰 만료, 서명 오류 | 자동 갱신, 서명 로직 모니터링 |
| Shopify | API 버전 분기별 업데이트 | 안정 버전 6개월 고정 후 마이그레이션 |
| 카카오톡 채널 | 템플릿 미승인, 발신 한도 초과 | 발송 전 템플릿 상태 확인, 레이트 리밋 가드 |
| 채널톡 | 토큰 갱신 없음 (키 유효기간 무제한) | 키 로테이션 반기 1회 권고 |

## 보안

- API 키: 환경 변수 또는 NCP Secret Manager에 저장
- 웹훅 수신 시 HMAC 서명 검증 필수
- 클라이언트 데이터: 자동화 처리 후 30일 이후 자동 삭제

---

## 면책 조항

- 플랫폼 자체(Cafe24, 네이버, Shopify 등)의 서비스 중단
- 카카오 알림톡 템플릿 심사 지연
- 클라이언트 측 인증 정보 변경 미공유
- 불가항력적 사유 (천재지변, 보안 침해 등)
