## 5. Tips & Best Practices

### On Making ClickUp + Claude Code Work Without n8n:

1. **The core problem stated explicitly**: Managed agents require an external API call to trigger — they can't self-schedule. This means true "no n8n" automation isn't fully native yet.

2. **Nate's preferred alternative to n8n**: **Trigger.dev**
   > *"I would just way rather use trigger.dev than use this managed agent environment. And it's also a lot cheaper."*
   - Trigger.dev lets you host code in the cloud, set up crons, delegate to different scripts

3. **Use ClickUp OAuth (not API keys) in managed agents**: 
   > *"It's kind of like an OAuth login... once you sign in, it gets stored in your vault."*
   - Vault can be shared across team members
   - Much easier than managing raw API keys

4. **Fix the system prompt before running**: Auto-generated system prompts are generic. Always do a **guided edit** to add specific business context before testing.
   > *"I would come over and I would click guided edit... This is important so that you can actually give me a good analysis."*

5. **Check your agent versions**: Managed agents create version histories — verify your edits actually saved.
   > *"You have to be really careful here because in the agents tab, we can see that I have two versions... They didn't even change. So, I don't think that it actually implemented that knowledge."*

6. **Use CLI to build richer agents**: Building managed agents via Claude Code CLI produces more robust system prompts than the quick-start UI.
   > *"What you'll notice is much more robust than the ones that the quick start version had us build."*

7. **ClickUp as output, not trigger (for now)**: Current architecture works best with ClickUp as a **destination** (comments, channel messages, summaries) rather than a reliable **trigger source** without additional tooling.

8. **Memory across sessions doesn't exist yet** (but is coming):
   > *"Each time the competitor intelligence agent wakes up again, it is completely stateless."*
   - Workaround: Have agent write logs/journals to files; next session reads those files

9. **Trading agent lesson**: Use **ClickUp for daily summaries** instead of Telegram if you "live in ClickUp more" — consolidates your AI outputs into your existing workflow hub.

10. **Avoid API keys in system prompts** when building via CLI:
    > *"Be very careful... it might try to default and send over potentially in the system prompt the API keys... just keep that in mind."*
