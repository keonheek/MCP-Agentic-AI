# @evolving.ai Style Guide — Static Baseline

Fallback reference when the live scrape (`evolving-ai-style-live.md`) fails. The live file takes priority if it exists and is from today.

## Hook (line 1)

One line. Either a sharp question or a bold claim.

Examples (Korean adapted):
- "이거 모르면 6개월 뒤에 후회함"
- "AI 에이전트가 진짜로 일을 하기 시작한 주"
- "오픈AI가 또 판을 뒤집었습니다"
- "왜 지금 이걸 알아야 할까요?"

## Body — 3 beats

1. **What happened** — 1-2 sentences, concrete event
2. **Why it matters** — 1-2 sentences, the implication
3. **What to do with it** — 1 sentence, takeaway or action

Line breaks between beats. No walls of text.

## CTA

One line at the end. Reuse patterns:
- "저장해두세요."
- "팔로우하고 매일 받아보세요."
- "더 궁금하면 댓글 남겨주세요."

## Hashtags

5-8 tags. Mix Korean and English.

Rotation pool:
- Korean: `#인공지능` `#AI뉴스` `#자동화` `#에이전트AI` `#LLM` `#생성형AI`
- English: `#AI` `#Agents` `#LLM` `#GenAI` `#MachineLearning` `#AINews`

## Formatting rules

- Line breaks between beats (blank line, not just newline)
- No emoji spam (0-2 max, only if they add meaning)
- No em dashes in captions (reserved for internal docs)
- Total length (hook + body, excluding hashtags): 80-150 Korean characters

## Visual pairing

- Feed post: 4:5 portrait
- Reel: 9:16 vertical
- Carousel: 1:1 square, 3-5 slides
- Strong text overlay on cover image (Korean headline 10-15 chars)

## Post format preference

Based on observed @evolving.ai output:
- ~50% Reels (15-30s)
- ~30% Carousels
- ~20% Single static posts

Routing rules:
- Source is a tweet with a clear visual hook → Reel or Carousel
- Source is a news article without strong visuals → Static or Carousel with text slides
