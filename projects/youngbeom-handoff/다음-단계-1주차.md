### 1. `/유튜브-주제` Skill

**파일 위치:** `.claude/commands/유튜브-주제.md`

```markdown
---
description: 유튜브 채널에 올릴 주제 아이디어 10개 생성
argument-hint: <채널 주제 또는 키워드>
allowed-tools: WebSearch
---

  - 왜 지금 이 주제가 먹히는지 이유

표 형식으로 정리해서 보여줘.
```

**테스트:** 터미널에 `/유튜브-주제 한국 경제` 치면 표가 나와야 함. 안 나오면 파일 이름 오타 체크.

## 화요일 — 첫 Sub-agent (1.5시간)

### `youtube-researcher`

**파일 위치:** `.claude/agents/youtube-researcher.md`

```markdown
---
name: youtube-researcher
description: 유튜브 채널 트렌드, 경쟁 채널, 조회수 패턴 리서치 전문
model: haiku
tools: WebSearch, WebFetch, mcp__youtube-transcript__get_transcript
---

너는 유튜브 리서치 전문가야.

맡은 일:
1. 최근 7일 이내 인기 유튜브 영상 찾기
2. 제목 패턴, 조회수, 댓글 반응 분석
3. 경쟁 채널이 어떤 주제로 성과 내는지 파악
4. 결과를 구조화된 마크다운으로 보고

보고 형식:
- 분석한 채널/영상 목록 (링크 포함)
- 핵심 인사이트 3개
- 주목할 트렌드
- 추천 행동 (다음에 뭐 만들지)

말 돌리지 말고 바로 결론부터.
```

**왜 haiku?** 리서치는 판단 많이 안 씀. 검색하고 요약만. Opus 쓰면 돈 아까움.

**왜 memory 안 넣음?** 리서치는 매번 새로 하는 게 맞음. 기억 쌓이면 옛날 데이터로 답함.

**테스트:** 메인 Claude한테 "youtube-researcher 써서 한국 경제 채널 3개 분석해줘" 시켜. 병렬로 도는지 확인.

## 수요일 — Notion MCP 연결 (1시간)

### 1. Notion 쪽 준비

1. notion.so/my-integrations 접속
2. "New integration" → 이름: Claude Code, 워크스페이스 선택
3. Internal Integration Token 복사 (secret_ 로 시작)
4. Notion 워크스페이스에서 Claude Code가 쓸 페이지 하나 만들고, 그 페이지 "…" 메뉴 → "Connections" → 방금 만든 integration 추가

이거 안 하면 MCP 연결돼도 Notion이 "페이지 못 찾음" 에러 냄.

### 2. `.mcp.json`에 추가

기존 파일 열어서 `mcpServers` 안에 추가:

```json
"notion": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-notion"],
  "env": {
    "NOTION_API_KEY": "secret_여기에_붙여넣기"
  }
}
```

### 3. VS Code 완전히 껐다 켜기

창 닫기 말고 **완전 종료**. MCP 변경은 재시작 없으면 반영 안 됨. (이거 저번에 건희도 까먹어서 30분 헤맸음)

### 4. 테스트

"Notion에 오늘 유튜브 주제 리스트 페이지 만들어줘" → 실제로 페이지 생기는지 확인.

## 목요일 — SessionStart Hook (30분)

매번 세션 열 때마다 "오늘 뭐 해야 돼?" 물어보기 귀찮잖아. Hook이 알아서 띄워줌.

### `.claude/settings.json` 수정

파일 없으면 만들고, 있으면 `hooks` 키 추가:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "type context\\current-priorities.md"
          }
        ]
      }
    ]
  }
}
```

**주의:** Windows라 `type`이고 경로는 `\\`임. Mac/Linux면 `cat context/current-priorities.md`.

기존 Stop hook이랑 같이 들어가야 되니까 병합할 때 조심.

**테스트:** 새 세션 열면 priorities 자동으로 뜸. 안 뜨면 JSON 문법 오류 (쉼표 빠진 거 99%).

이게 이번 주 진짜 목표. 앞에 만든 거 다 써서 영상 1개 기획부터 스크립트까지 감.

### 순서

1. `/유튜브-주제 이번 주 한국 뉴스`
   → 10개 중 하나 고름

2. youtube-researcher한테 경쟁 채널 3개 분석 맡김
   → "[고른 주제]로 한국 유튜브에 누가 비슷한 거 냈는지, 조회수 어떻게 나왔는지 분석해줘. youtube-researcher 써서."

3. `/스크립트-초안 [고른 주제]`
   → 리서치 결과 보고 한 번 더 다듬으라고 시킴

4. "이거 Notion에 '[주제명]' 제목으로 저장해줘"
   → Notion MCP가 저장

5. `/end`로 마무리

### 성공 기준

- 주제 선택부터 Notion 저장까지 **30분 이내**
- 중간에 안 막히고 한 번에 감
- 저장된 스크립트 그대로 찍을 수 있을 정도

중간에 막히면 거기가 다음 주 고쳐야 할 부분이야. 막히는 게 정상.

## 막혔을 때 (건희한테 물어보기 전에)

1. **Skill이 안 뜸** → 파일명 확인. `.md` 붙었는지. 폴더 위치 맞는지 (`commands/` vs `skills/`).
2. **MCP 툴 못 찾음** → VS Code 완전 재시작. Task Manager에서 프로세스까지 확인.
3. **Sub-agent가 main Claude 같은 답을 함** → frontmatter `model: haiku` 있는지. tools 제한 걸었는지.
4. **Hook이 안 돎** → `settings.json` JSON 문법 체크 (jsonlint.com에 붙여넣기).
5. **Notion "페이지 못 찾음"** → Integration을 해당 페이지에 추가 안 함. 위 수요일 1번 다시.
